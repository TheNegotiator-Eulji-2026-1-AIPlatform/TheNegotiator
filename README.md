# 데이터 전처리 과정

## 1. 원본 데이터셋 구성

AIHub 감성대화말뭉치 데이터셋을 사용하였다.
원본 데이터셋은 다음과 같은 컬럼으로 구성되어 있다.

| 컬럼명    | 설명              |
| ------ | --------------- |
| 감정_대분류 | 상황의 감정 유형       |
| 사람문장1  | 사용자의 감정 상황 문장   |
| 시스템문장1 | 상황에 대한 공감 응답 문장 |

감정_대분류는 다음 6가지로 구성되어 있었다.

* 분노
* 불안
* 슬픔
* 상처
* 당황
* 기쁨

---

# 2. 기쁨 데이터 제거

프로젝트 목표는 “부정적인 감정을 가진 AI를 위로하여 안정시키는 시스템”이기 때문에, 긍정 감정인 “기쁨” 상황은 학습 대상에서 제외하였다.

따라서 다음 감정만 사용하였다.

* 분노
* 불안
* 슬픔
* 상처
* 당황

---

# 3. 학습 데이터 구조 변경

기존 데이터셋은 단순히 감정 상황과 시스템 응답으로 구성되어 있었지만, 프로젝트에서는 사용자의 입력이 현재 감정 상황에 적절한 위로인지 판단해야 했다.

따라서 데이터를 다음과 같은 구조로 재구성하였다.

| ai_text   | player_input | label |
| --------- | ------------ | ----- |
| AI의 감정 상황 | 사용자의 입력 문장   | 감정 라벨 |

예시:

| ai_text | player_input | label   |
| ------- | ------------ | ------- |
| 너무 외로워  | 많이 힘드셨겠어요    | sadness |

---

# 4. 감정 라벨 재구성

모델이 “어떤 감정 상황에 적절한 위로인지” 구분할 수 있도록 감정명을 영어 라벨로 변환하였다.

| 기존 감정 | 변환 라벨         |
| ----- | ------------- |
| 분노    | anger         |
| 불안    | anxiety       |
| 슬픔    | sadness       |
| 상처    | hurt          |
| 당황    | embarrassment |

---

# 5. 부정 응답 데이터 생성

기존 데이터셋에는 공감 문장만 존재했기 때문에, 모델이 “위로가 되지 않는 문장”을 학습할 수 있도록 부정 응답 데이터를 직접 추가하였다.

추가한 부정 문장은 다음 유형으로 구성하였다.

* 비난형
* 무시형
* 냉소형
* 회피형
* 맥락 무시형
* 의미 불명형

예시:

* “어쩌라고”
* “그건 네 잘못이잖아”
* “관심 없어”
* “오늘 점심 뭐 먹었어?”
* “ㅁㄴㅇㄹ”

---

# 6. BAD 데이터 생성 방식

각 감정 상황(ai_text)에 대해 랜덤한 부정 문장 3개를 연결하여 BAD 데이터를 생성하였다.

예시:

| ai_text | player_input | label |
| ------- | ------------ | ----- |
| 너무 외로워  | 어쩌라고         | bad   |
| 너무 외로워  | 관심 없어        | bad   |
| 너무 외로워  | 오늘 점심 뭐 먹었어? | bad   |

이를 통해 모델이 공감 응답과 비공감 응답을 구분할 수 있도록 하였다.

---

# 7. 최종 데이터셋 구성

최종적으로 데이터셋은 다음 6개의 클래스로 구성되었다.

* anger
* anxiety
* sadness
* hurt
* embarrassment
* bad

---

# 8. 데이터 셔플 및 저장

생성된 데이터를 무작위로 섞은 후 다음 파일로 저장하였다.

| 파일명       | 용도    |
| --------- | ----- |
| train.csv | 모델 학습 |
| test.csv  | 모델 평가 |

저장 형식은 UTF-8-SIG 인코딩을 사용하였다.
